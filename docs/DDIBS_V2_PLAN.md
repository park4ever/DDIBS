# DDIBS V2 Plan

## 1. 문서 목적

이 문서는 DDIBS V2에서 무엇을 진행할지 간단히 정리한 계획 문서다.

V1이 핵심 플로우 완성과 정합성·동시성·상태 전이 증명에 집중한 버전이었다면,
V2는 **운영 안정성, 테스트 신뢰도, 설명 가능성**을 높이는 단계로 정의한다.

이 문서는 Blueprint처럼 큰 기준 문서가 아니라,
**V2에서 어디까지 진행하고 무엇을 제외할지 빠르게 합의하기 위한 실무형 메모**에 가깝다.

---

## 2. V2 목표

V2의 목표는 기능 수를 늘리는 것이 아니라,
이미 완성한 핵심 플로우를 **더 안정적이고, 더 검증 가능하고, 더 설명 가능하게 만드는 것**이다.

핵심 목표는 아래 3가지다.

1. 배치/운영 관점의 안정성 강화
2. 테스트 환경과 검증 신뢰도 강화
3. 관리자 조회 및 문서/시연 완성도 강화

---

## 3. V2 방향 요약

### V2에서 집중할 것
- 배치 멱등성 / 중복 실행 관점 정리
- 테스트 환경 고도화
- 경합/상태 전이 검증 보강
- 관리자 조회 기능 고도화
- 시연/문서 설명력 강화

### V2에서 지금 하지 않을 것
- 장바구니
- 쿠폰 / 포인트
- 리뷰 / 문의
- 배송 / 송장
- 환불 / 교환
- 실제 PG 연동
- 과한 프론트엔드 확장
- 마이크로서비스 / Kafka / K8s

즉 V2는 **범용 커머스 기능 확장판이 아니라,
고수요 발매 백엔드 완성도 강화판**으로 간다.

---

## 4. In Scope

### 4.1 배치 안정성 고도화
우선순위: 상

목표:
- 정산 생성 배치의 멱등성 구조 정리
- 홀드 만료 배치의 중복 실행 / 재처리 관점 점검
- 배치 실행 로그와 결과 요약 정리
- 실패 시 재실행 전략을 문서 수준에서라도 명확히 정리

예상 작업:
- SettlementBatchService 점검
- HoldExpirationBatchService 점검
- 스케줄러 로그 개선
- 트러블슈팅 문서 보강

### 4.2 테스트 환경 고도화
우선순위: 상

목표:
- 로컬 MySQL 의존도를 줄이고 테스트 재현성을 높인다
- 실제 MySQL 환경과 더 가까운 방식으로 테스트를 강화한다

예상 작업:
- Testcontainers + MySQL 적용 검토
- 핵심 통합 테스트 전환 또는 병행 구성
- 경합 테스트 반복 실행 전략 검토
- 상태 전이 관련 테스트 보강

### 4.3 관리자 조회 고도화
우선순위: 중

목표:
- 운영자가 발매 / 주문 / 정산을 더 실용적으로 조회할 수 있도록 개선한다

예상 작업:
- 주문 조회 조건 추가
- 정산 조회 조건 추가
- 기간 / 상태 / 판매자 기준 검색
- 페이징 / 정렬 정리

### 4.4 문서 / 시연 고도화
우선순위: 중

목표:
- DDIBS를 면접 / 포트폴리오 관점에서 더 설명 가능하게 만든다

예상 작업:
- 시퀀스 다이어그램 추가
- API 흐름 정리
- 트러블슈팅 문서 보강
- 시연 시나리오 정리
- README 보강

---

## 5. Out of Scope

아래 항목은 V2 범위에 넣지 않는다.

- 장바구니
- 쿠폰 / 포인트
- 리뷰 / 문의
- 배송 / 송장
- 환불 / 교환
- 외부 PG 실제 연동
- 사용자용 화려한 프론트엔드
- 마이크로서비스 분리
- Kafka / MQ
- K8s

이 항목들은 DDIBS의 핵심 메시지를 흐릴 수 있으므로,
최소 V2에서는 제외한다.

---

## 6. 권장 진행 순서

### Step 1. 배치 안정성 점검 및 보강
- 정산 생성 배치 점검
- 홀드 만료 배치 점검
- 멱등성 / 재처리 관점 정리

### Step 2. 테스트 환경 고도화
- Testcontainers 도입 여부 결정
- 핵심 테스트 개선
- 경합 테스트 보강

### Step 3. 관리자 조회 기능 개선
- 주문 / 정산 / 발매 조회 조건 추가
- 검색/필터/페이징 정리

### Step 4. 문서 및 시연 보강
- 시퀀스 다이어그램
- 트러블슈팅
- 시연 스크립트
- README 보강

---

## 7. V2 완료 기준

V2는 아래 조건을 만족하면 마감으로 본다.

- 배치/운영 관점에서 현재 구조를 설명 가능하다
- 테스트 환경이 V1보다 더 재현 가능하고 신뢰도 있게 정리된다
- 관리자 조회 기능이 최소 한 단계 실용적으로 개선된다
- 문서/시연 자료가 V1보다 더 명확해진다

즉 V2는 새로운 커머스 기능을 많이 넣는 것이 아니라,
**현재 DDIBS를 더 실무적으로 보이게 만드는 단계**로 마감한다.

---

## 8. V3 예고

V2 이후 V3에서는 아래를 검토할 수 있다.

- 관리자/운영 화면 중심의 시연 완성도 강화
- API/조회 흐름 고도화
- 프론트엔드 데모 레이어 최소 도입

단, V3에서도 중심은 여전히 백엔드이며,
프론트엔드는 DDIBS 백엔드를 설명하기 위한 보조 수단으로 다룬다.

---

## 9. 한 줄 결론

DDIBS V2는 기능 추가 중심 단계가 아니라,
**배치 안정성 · 테스트 신뢰도 · 관리자 조회 · 문서/시연 완성도**를 높여
고수요 발매 백엔드 프로젝트로서의 완성도를 끌어올리는 단계다.
